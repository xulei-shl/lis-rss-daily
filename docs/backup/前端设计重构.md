# LIS-RSS Literature Tracker - 前端重构设计规范 (v2.0)

**版本**：2.0 (Modern Ethereal 风格重构版)  
**适用对象**：前端开发、UI 设计师  
**核心目标**：将原有的“学术 Brutalist（粗野主义）”风格重构为“**现代轻盈学术（Modern Ethereal Academia）**”风格。

---

## 一、 设计核心理念

**关键词**：`柔和`、`悬浮`、`衬线优雅`、`暖色点缀`

1.  **去边框化 (De-bordering)**：摒弃原有的 1px 硬实线边框，改用**大面积留白**和**弥散阴影**来区分层级。
2.  **柔性容器 (Soft Containers)**：所有的卡片、按钮、输入框均采用**大圆角**，拒绝直角。
3.  **排版艺术 (Typographic Contrast)**：标题使用**高雅的衬线体**（且支持斜体强调），正文保持现代无衬线体，形成古典与现代的视觉张力。
4.  **暖色引导 (Warm Accents)**：主色调由原来的“冷峻蓝”改为“**暖橙色**”，营造更具沉浸感和温度的阅读环境。

## 二、 设计 Token (Design Tokens)

请直接使用以下 CSS 变量覆盖原有的设计系统。

### 2.1 颜色系统 (Color Palette)

摒弃纯黑纯白，使用高级灰与暖色调。

```css
:root {
  /* --- 基础背景 --- */
  /* 页面整体背景：浅暖灰，类似未漂白的再生纸或石膏色 */
  --bg-primary: #F2F2F2;
  /* 卡片/内容表面：纯白 */
  --bg-surface: #FFFFFF;
  /* 悬浮/磨砂玻璃背景：带透明度的白 */
  --bg-glass: rgba(255, 255, 255, 0.75);

  /* --- 文字系统 --- */
  /* 主标题：深炭黑，非纯黑 */
  --text-primary: #1A1A1A;
  /* 正文/次要信息：中性灰 */
  --text-secondary: #666666;
  /* 辅助/占位符：浅灰 */
  --text-tertiary: #999999;

  /* --- 品牌与功能色 --- */
  /* [核心改变] 品牌主色：活力暖橙 - 用于高亮、CTA按钮、关键数据 */
  --accent-primary: #FF9F43;
  /* 品牌色悬停态 */
  --accent-hover: #FF8F20;
  /* 品牌色背景上的文字 */
  --accent-text: #FFFFFF;
  /* 品牌色的浅色背景（用于标签/次要高亮） */
  --accent-subtle: rgba(255, 159, 67, 0.15);

  /* --- 状态色 (保持语义，调整饱和度) --- */
  --status-green: #2ECC71; /* 通过 */
  --status-red: #FF6B6B;   /* 拒绝 */
  --status-blue: #54A0FF;  /* 处理中 */
}
```

### 2.2 字体排版 (Typography)

**极重要**：风格转型的关键在于标题字体的更换。

*   **标题字体 (Heading)**：必须使用**衬线体 (Serif)**。
    *   *推荐字体*：`Playfair Display` (Google Fonts) 或 `Merriweather`。
    *   *样式特征*：标题中应支持使用 `<em>` 或 `<i>` 标签，样式表现为**斜体 (Italic) 且颜色为品牌橙色**，用于强调关键词。
*   **正文字体 (Body)**：保持**无衬线体 (Sans-serif)**。
    *   *推荐字体*：`Inter`, `Plus Jakarta Sans` 或 `System UI`。

```css
:root {
  --font-heading: 'Playfair Display', serif; /* 关键变化 */
  --font-body: 'Inter', sans-serif;
}

/* 全局标题样式重置 */
h1, h2, h3, h4 {
  font-family: var(--font-heading);
  font-weight: 700;
  letter-spacing: -0.02em; /* 稍微收紧字间距，更显精致 */
  color: var(--text-primary);
}

/* 标题中的强调词样式 - 核心视觉特征 */
h1 em, h2 em {
  font-family: var(--font-heading);
  font-style: italic;
  color: var(--accent-primary); /* 橙色斜体 */
  font-weight: 400; /* 斜体通常字重稍轻 */
}
```

### 2.3 形状与质感 (Shape & Depth)

从“硬朗直角”转变为“柔和圆角”。

```css
:root {
  /* --- 圆角系统 --- */
  /* 大卡片/容器圆角 */
  --radius-lg: 24px;
  /* 输入框/小卡片圆角 */
  --radius-md: 16px;
  /* 按钮/标签圆角 (胶囊形) */
  --radius-full: 9999px;

  /* --- 阴影系统 (弥散光感) --- */
  /* 默认悬浮状态：大模糊半径，低透明度，向下位移 */
  --shadow-card: 0 10px 40px -10px rgba(0, 0, 0, 0.08);
  /* 悬停状态：阴影加深，位移增加 */
  --shadow-hover: 0 20px 60px -15px rgba(0, 0, 0, 0.12);
  /* 橙色光晕：用于强调按钮 */
  --shadow-accent: 0 8px 20px -4px rgba(255, 159, 67, 0.5);
  
  /* --- 动效 --- */
  --transition-smooth: all 0.4s cubic-bezier(0.25, 0.8, 0.25, 1);
}
```

## 三、 核心组件重构规范

### 3.1 导航栏 (Navbar)
*   **原有**：顶部实色背景条。
*   **重构**：**透明磨砂玻璃悬浮顶栏**。
    *   背景：`backdrop-filter: blur(12px)` + `background: rgba(255,255,255,0.8)`。
    *   位置：`sticky top: 0`。
    *   Logo：左侧，纯黑色文字，衬线体。
    *   链接：右侧，纯黑色文字，无衬线体，Hover 时出现橙色下划线。

### 3.2 按钮 (Buttons)
*   **主按钮 (Primary)**：
    *   形状：**全圆角胶囊形** (`border-radius: 999px`)。
    *   背景：纯黑色 (`#000000`) 或 品牌橙色 (`var(--accent-primary)`)。
    *   文字：白色。
    *   阴影：带有轻微的弥散阴影。
*   **次要按钮 (Secondary)**：
    *   形状：全圆角胶囊形。
    *   背景：透明。
    *   边框：1px 浅灰边框 (`#E5E5E5`)。
    *   文字：深灰色。

### 3.3 文章卡片 (Article Card) - **视觉重点**
*   **容器**：纯白背景，`--radius-lg` (24px) 圆角，**无边框**，应用 `--shadow-card`。
*   **布局**：
    1.  **顶部**：左侧显示来源（灰色小胶囊标签），右侧显示时间（灰色小字）。
    2.  **标题**：大号衬线体 (`font-family: var(--font-heading)`)，字号 `1.5rem`，行高 `1.3`。
    3.  **摘要**：无衬线体，灰色，行高 `1.6`，字间距舒适。
    4.  **底部栏**：
        *   关键词：使用 `#` 号前缀，灰色文字，水平排列。
        *   **操作区**：如果是“推荐/通过”的文章，右下角放置一个**橙色圆形图标按钮**（带箭头），作为点击进入详情的诱导。
*   **交互**：鼠标 Hover 时，卡片整体上浮 (`transform: translateY(-5px)`)，阴影变为 `--shadow-hover`。

### 3.4 搜索框 (Search Input)
*   **原有**：标准矩形输入框。
*   **重构**：**超大号对话框式输入**。
    *   高度：`60px` 以上。
    *   圆角：`--radius-full` (胶囊形) 或 `--radius-lg`。
    *   背景：纯白。
    *   阴影：内阴影或轻微外阴影。
    *   Placeholder：居中或左对齐，字号较大，提示语如 "Ask me anything about literature..."。
    *   按钮：输入框内部右侧内嵌一个圆形的“搜索”图标按钮。

### 3.5 统计模块 (Stats Dashboard)
*   **原有**：顶部四个方块卡片。
*   **重构**：**深色悬浮底座 (Dark Dock)**。
    *   将统计数据整合为一个横向的长条容器。
    *   背景：深灰色 (`#2D2D2D`) 或 纯黑。
    *   文字：白色。
    *   数字：使用品牌橙色 (`var(--accent-primary)`) 高亮显示。
    *   圆角：`--radius-lg`。
    *   位置：可置于 Hero 区域下方，或者作为页面底部的 Sticky Bar。

## 四、 页面级布局变更

### 4.1 首页 (index.ejs)
*   **新增 Hero 区域**：
    *   在文章列表上方增加一个宽大的头部区域。
    *   **文案**：主标题使用超大号衬线体 (e.g., 64px)，例如 "Turn Knowledge into *Insight*"，其中 "Insight" 使用橙色斜体。
    *   **描述**：下方配一行灰色副标题。
*   **文章流**：
    *   取消原本的“表格线”或“日期分割线”。
    *   日期显示在左侧空白处（Sticky 定位），右侧展示该日期的文章卡片流，形成**时间轴 (Timeline)** 的感觉。

### 4.2 详情页 (article-detail.ejs)
*   **阅读模式**：
    *   页面背景保持浅灰 (`--bg-primary`)。
    *   正文区域设计为一张**居中的纯白“纸张”**。
    *   宽度限制：`700px - 800px` (最佳阅读宽度)。
    *   内边距：`60px` 以上，留足呼吸感。
*   **AI 增强区**：
    *   AI 生成的摘要、翻译、分析内容，不使用分割线，而是使用**浅橙色背景块** (`--accent-subtle`) 包裹，圆角 `16px`，以此与原文区分。

### 4.3 登录页 (login.ejs)
*   **布局**：左右分栏或居中悬浮卡片。
*   **视觉**：
    *   背景：极简灰。
    *   卡片：纯白，大圆角，弥散阴影。
    *   左侧（如果是分栏）：可以是一张抽象的艺术线条图或纯品牌橙色色块。
    *   表单：输入框高度加大，去边框，使用浅灰背景 (`#F7F7F7`)。

## 六、 总结：前端开发 Checklist

1.  [ ] **引入字体**：在 `layout.ejs` 头部引入 Google Fonts (`Playfair Display` 和 `Inter`)。
2.  [ ] **替换变量**：将 `style.css` 中的颜色和圆角变量全部替换为本文档第二节的内容。
3.  [ ] **去除边框**：全局搜索 `border: 1px solid`，除输入框 focus 态外，基本都应移除或替换为 `box-shadow`。
4.  [ ] **增加留白**：将列表项的 `margin-bottom` 从原来的 `16px` 增加到 `32px` 或 `40px`。
5.  [ ] **重写标题**：检查所有 `h1`-`h3` 标签，确保应用了衬线体样式。
