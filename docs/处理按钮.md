问题 1: /articles 页面的批量处理是什么？
答案：是大模型总结，不是文章过滤。

批量处理执行的是完整的 AI 处理流程，包含三阶段 Pipeline：

处理流程 (pipeline.ts:241-360)
阶段	功能
Stage 1: 抓取	使用 Playwright 抓取文章全文，转换为 Markdown 格式
Stage 2: 分析	调用 LLM 生成：中文摘要、标签、研究洞察、相关文章
Stage 3: 导出	将文章导出为 Markdown 文件到 data/exports/ 目录
API 调用链

前端 processBatch() 
  → POST /api/articles/process-batch
    → processBatchArticles()
      → processArticle() [每篇文章]
        → runPipeline() [三阶段处理]
重要约束
只处理 filter_status='passed' 的文章（见 pipeline.ts:94）
如果文章是 pending 或 rejected 状态，会被跳过
问题 2: 文章详情页的"重新处理"按钮执行什么逻辑？
答案：触发与批量处理相同的完整 AI Pipeline。

按钮显示逻辑
按钮只在以下状态显示：

process_status = 'pending'（待处理）
process_status = 'failed'（处理失败）
API 调用

点击"重新处理"
  → POST /api/articles/:id/process
    → processArticle(articleId)
      → runPipeline() [三阶段处理]
关键代码 (article-process.ts:39-71)

// 后台异步处理，不阻塞响应
processArticle(articleId, userId)
  .then((result) => { log.info(...) })
  .catch((error) => { log.error(...) });

res.json({ success: true, message: 'Article processing started' });
总结对比
功能	触发方式	处理范围	执行逻辑
批量处理	文章列表页"批量处理"按钮	多篇文章（默认 10 篇）	三阶段 Pipeline（抓取+分析+导出）
重新处理	文章详情页"重新处理"按钮	单篇文章	三阶段 Pipeline（抓取+分析+导出）
两者调用的是同一套处理逻辑 (processArticle())，区别只是单篇 vs 批量。
